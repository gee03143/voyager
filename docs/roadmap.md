# Voyage 개발 로드맵

> 계획 기준: 주 5일 · 하루 ~5 집중시간 (가용 8시간 중 슬랙 반영) ≈ **약 4 작업일/주**
> 아트는 전부 플레이스홀더 전제. 진짜 아트·애니메이션·폴리시는 별도.
> 이 문서는 살아있는 계획 — 막히면 뒤로 밀고 버퍼 주가 흡수한다.

## 시계 탭 구조 (설계)

```
시계 탭
├── 세션 (포모도로)   ← Pomodoro(Countdown)   : 집중/휴식 단계별 카운트다운
│     설정: 집중 시간, 짧은 휴식, 긴 휴식, 반복 사이클 수
├── 타이머            ← Countdown 직접          : 정해진 시간 뒤 알람 (1회)
├── 알람              ← 별도 부품 AlarmClock     : 목표 "벽시계 시각" 도달 감지
│     (요일/반복 로직은 데일리 체크리스트와 재사용)
└── 설정              ← 지속 전역 설정 (Autoload, 단일 진실)
	  ├ 카운트다운 보이기/숨기기   (뷰)            — 즉시 가능
	  ├ 타이머 스타일 Timeline/Circle (뷰 교체)    — 즉시 가능
	  ├ 소리 설정                 (오디오)         — 오디오 노드 필요
	  └ 활성화 시 자동 최소화      (컴패니언 동작)   — Week 7 의존
```

설계 원칙:
- **세션·타이머는 Countdown 부품 공유. 알람은 별도 메커니즘** (지속시간 카운트다운이 아니라 시각 감지).
- **타이머 스타일은 뷰 문제** — 로직은 `ticked` 만 쏘고 표시 위젯(Timeline/Circle)을 갈아끼움. 스타일 분기를 타이머 로직에 넣지 않는다.
- **설정값은 전부 데이터 계층(Autoload)에 모은다** — 시계 뷰와 컴패니언이 같은 곳을 읽는다.
- **자동 최소화**는 설정 항목만 먼저 정의, 실제 동작은 컴패니언(Week 7) 연결.
- 빌드 순서: **세션 → 데이터 계층 → 설정/알람** (설정은 저장할 곳이 있어야 의미 있음).

## MVP — 7주

### Week 1 · 타이머 완성 + 데이터 계층 착수 — ✅ 완료
- [x] 포모도로 (유한 세션 계획 · skip · 타임라인 칩 · 인라인 설정 · save-on-start)
- [x] 일반 타이머 (시/분/초 입력 · 영속화)
- [x] Autoload `Save` + 저장/불러오기 (JSON, `user://save.json`, version 필드)
- ➕ 계획 초과: 알람 탭(CRUD·발화·소리), 설정 탭, ClockTab 커스텀 nav, `ClockToolView` 공통 베이스
- ⚠️ 목표의 "총 집중시간 persist" 는 미완 — `focus_finished` 가 아직 누적 안 함 → **Week 4(항해 진행)** 소관 (영속화 인프라는 준비됨)

### Week 2 · 데이터 계층 마무리 + Todo 기초

## 메모

- 슬랙 포함 추정. 마지막 20%(버그·연결·세이브 안정성·전환 처리)는 항상 늘어남.
- 가장 위험한 주: **Week 3(드래그·날짜 로직)**, **Week 7(윈도우 다루기)**.
- 데이터 계층(Week 1~2)은 한 번 깔면 이후 모든 기능을 빠르게 함 — 투자 우선순위 높음.

### 미룬 항목 (계획엔 있으나 후순위)
- **타이머 스타일 Timeline/Circle**: 현재 Timeline(바)만. Circle 위젯은 폴리시/별도 작업.
- **활성 시 자동 최소화**: 설정·플래그·print 훅까지 완료. 실제 미니 모드 전환은 **Week 7(컴패니언)**.
- **총 집중시간 추적·영속화**: **Week 4(항해 진행)** 에서 `focus_finished` 연결 시.
- **알람 전역화**: 현재 `AlarmClock` 이 알람 뷰 소유. 앱 셸 생기면 autoload 로 올려 전역 발화 고려.
- **리스트 패턴 추출**: AlarmRow/AlarmView 의 추가/삭제 구조 → Todo 만들 때 공통 베이스로 추출.
